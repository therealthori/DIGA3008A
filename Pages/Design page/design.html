Design Documentation

UI/UX Process & Design System

Introduction

This design section provides comprehensive documentation of the Interaction Design (IxD) process undertaken for ThoriStory. The design choices and accessibility features are guided by fundamental usability principles of simplicity, readability, and effective communication. Each design decision has been specifically crafted to provide optimal functionality and user experience while maintaining visual appeal and brand consistency.

Design Philosophy

The design approach emphasizes user-centered design with a focus on creating an intuitive, accessible, and visually compelling digital experience. The aesthetic challenges conventional web design through the use of unorthodox styles, visual gradients, and encompassing imagery while maintaining usability standards.

Goal Alignment

The primary objectives driving the design decisions are centered around creating a cohesive and functional user experience:

Enhance Usability

Create an interface that is easy to use, intuitive, and efficient for users to perform their tasks without confusion or unnecessary complexity.

Promote Consistency

Maintain coherent structure, layout, and interaction patterns across all pages, reducing learning time and cognitive load for users.

Encourage Learnability

Design interfaces that users can quickly understand and master, even on first use, through familiar patterns and clear visual hierarchy.

Facilitate Feedback

Provide timely and meaningful feedback that helps users understand system status and results of their actions through visual cues and interactions.

Structural Organization

The folder structure follows a logical hierarchy that separates content types, making the website maintainable and scalable. Each major section (Portfolio, Academia, Design, About) has its dedicated directory with associated assets.

User Flow - Screens, Behaviour, and Decisions

User flow mapping showing primary navigation paths and decision points

Navigation Strategy

The user flow is designed to provide multiple entry points while maintaining a clear hierarchical structure. Users can access any major section from the main navigation, with contextual links providing cross-references between related content.

Primary User Journeys

  • Portfolio Discovery: Home → Great Works → Individual Projects → Contact
  • Academic Content: Home → Academia → Blog Posts → Related Articles
  • Design Process: Home → Designs → Process Documentation → Style Guide
  • Personal Connection: Home → The Boi → Social Media → Contact

Interface Elements

Navigation System

  • Fixed Header: Persistent navigation for consistent access to main sections
  • Logo/Brand: "ThoriStory" serves as home link and brand identifier
  • Menu Items: Clear, descriptive labels with hover states for feedback
  • Back-to-Top Button: Floating action button for long-form content pages

Content Organization

  • Highlight Boxes: Emphasized content sections with colored borders
  • Image Galleries: Responsive grid layouts for visual content
  • Section Dividers: Visual separation between content blocks
  • Interactive Elements: Hover effects and smooth transitions

Composition & Visual Hierarchy

Hierarchy Principles

  • Scale: Progressive font sizes create clear information hierarchy
  • Color: Strategic use of brand colors to guide attention
  • Spacing: Consistent margins and padding for visual breathing room
  • Contrast: Sufficient color contrast for accessibility compliance

Balance & Harmony

  • Grid System: Flexible responsive grid for consistent alignment
  • White Space: Strategic use of negative space for visual clarity
  • Proportions: Golden ratio principles in layout construction
  • Visual Weight: Balanced distribution of elements across pages

Accessibility & Usability Considerations

Accessibility Features

  • Color Contrast: WCAG AA compliant color combinations
  • Font Sizing: Scalable typography using relative units
  • Keyboard Navigation: Full keyboard accessibility for all interactive elements
  • Alt Text: Descriptive alternative text for all images
  • Semantic HTML: Proper heading hierarchy and semantic markup

Responsive Design

  • Mobile-First: Design optimized for mobile devices first
  • Flexible Grids: CSS Grid and Flexbox for adaptive layouts
  • Touch Targets: Minimum 44px touch targets for mobile interaction
  • Performance: Optimized images and efficient CSS for fast loading

Future Iterations

Planned Enhancements

  • Enhanced animation and micro-interactions for improved user engagement
  • Advanced filtering and search functionality for portfolio and blog content
  • Progressive Web App (PWA) features for offline accessibility
